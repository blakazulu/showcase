# Israel / ATS CV rules (distilled scorecard)

Distilled from `docs/cv_report.md` (which carries the full sourcing: Greenhouse,
Lever, Workday, Taleo, MIT, Drushim, AllJobs, Israeli law/privacy). Use this as the
Phase 2 checklist and the Phase 3 guardrails. When you need the "why" or a citation,
open `docs/cv_report.md`.

## The one-sentence strategy

Clean, one-column, text-first CV; page one carries the full case for interview
selection; wording mirrors the target posting *where true*. Semantic clarity plus
exact relevance beats both keyword-stuffing and clever manipulation.

## ATS format - do

- One column, top to bottom. Standard headings: Summary, Experience, Education,
  Skills, Languages, Projects, Military Service, Certifications.
- DOCX or text-based (selectable) PDF. Keep files well under 2.5 MB.
- Full, spelled-out job titles ("Senior Software Engineer", not "Sr. SW Eng").
- Contact details as plain text at the top of the page body.
- Restrained bold for hierarchy; one neutral font (Calibri/Arial/Georgia/Helvetica),
  >= 10 pt; generous white space; line length ~50-75 chars.

## ATS format - avoid (these break parsers)

- Columns, tables, text boxes, headers/footers containing contact info.
- Graphics, icons, photos, infographics, heavy color, dense design-template styling.
- Image-only / scanned PDFs (unparseable).
- Abbreviated job titles and abbreviated relevant skills.
- Interleaved Hebrew/English in a grid; mixed reading direction on one line.
- Hidden text, white-on-white keywords, prompt-injection tricks (mistrust/rejection).

## Israeli privacy - omit by default

Drop unless a formal state/academic process explicitly requires it: date of birth /
birth year, marital & family status, ID number, photo, religion, reserve-duty
details, salary expectations. Israeli anti-discrimination + privacy law make these
legally awkward and they don't help selection. Provide identity/credential docs only
when the process formally demands them (some public-sector, MFA, academic tracks).

## Content quality

- **One target title** under the name - don't make the recruiter infer direction.
- **Summary:** 2-4 lines, employer's vocabulary, years + domain + 2-3 core strengths
  + one concrete differentiator.
- **Bullets:** `action verb + scope + method + outcome/metric`.
  - Weak: "Responsible for backend services."
  - Strong: "Built payment-reconciliation microservices handling 2.1M monthly
    events, cutting manual finance review time by 38%."
- **Relevance:** include only experience that supports the target role; irrelevant
  history dilutes fit. Keep separate tailored versions per job family.
- **Projects section** when it proves hands-on ability better than another job bullet
  - strong for Israeli tech/product/data/design roles that ask for GitHub/portfolio.
- **Length:** one page preferred; two pages allowed for senior/multinational roles
  only if page one already earns the interview.

## Localization (Israel)

- Maintain **two separate masters**: English + Hebrew. Don't ship one interleaved
  bilingual file.
- English for international/hi-tech/multinational/many-finance roles; Hebrew for
  local commercial/operational/public-facing/NGO roles.
- Hebrew CV: fully Hebrew, English technical terms in normal spelling inside a
  labeled Skills line, one direction per line, consistent dates, plain-text contact
  block at top.

## Keyword method (4 steps)

1. Extract hard skills, titles, tools, certs, domain nouns from the posting.
2. Mirror the employer's phrasing without copying whole sentences.
3. Put the top terms in searchable zones: headline, summary, recent roles, projects,
   skills.
4. Prefer full standard titles and spelled-out terms over internal abbreviations.

## Pre-send checklist

- [ ] Single target title under the name.
- [ ] 2-4 line summary in the role's vocabulary.
- [ ] Every bullet is action + scope + method + metric where possible.
- [ ] One column, standard headings, no tables/text boxes/graphics/columns.
- [ ] Full titles and skills spelled out.
- [ ] No birth year / marital status / ID / photo / reserve-duty / salary.
- [ ] Contact info is plain text at the top, not in a header/footer.
- [ ] English and Hebrew are separate files, each single-direction-per-line.
- [ ] Exported text (TXT/DOCX dump) keeps contact info and linear reading order.
- [ ] Mobile-readable: no horizontal scroll / tiny text on page one.
- [ ] No em-dashes, en-dashes, or emojis.
